# Jocker Bot 🃏

Telegram-бот, который шутит каждые N сообщений. Выбирает лучшую шутку с помощью LLM, подстраиваясь под контекст чата.

## Быстрый старт

```bash
cp .env.example .env
# Заполни BOT_TOKEN, LLM_API_KEY
npm install
npm start
```

## Подготовка данных

Положи файл с анекдотами в `data/jokes.json`. Формат:

```json
[
  {
    "text": "Текст анекдота",
    "tags": ["тег1", "тег2"],
    "source": "откуда"
  }
]
```

Пример — `data/jokes.example.json`.

## Как работает

1. Каждые 100 сообщений (настраивается) бот ищет шутку.
2. Ищет по RAG в твоём файле анекдотов — по ключевым словам из контекста.
3. Генерирует свежие шутки через LLM (DeepSeek / OpenAI).
4. LLM оценивает все варианты — выбирает лучший.
5. Если оценка ниже порога — пропускает до следующего раза.
6. Запоминает уже показанные шутки (избегает повторов).
7. Собирает профиль чата: кто активен, на какие шутки реагируют.

## Настройка

Всё в `.env`:
- `JOKE_INTERVAL` — каждые N сообщений
- `QUALITY_THRESHOLD` — порог оценки (0-10)
- `DEDUP_WINDOW` — сколько последних шуток не повторять
- `PROBE_WINDOW` — сколько сообщений ждать, если шутка не прошла

## Переменные окружения

| Переменная | По-умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | — | Токен Telegram-бота (обязательно) |
| `CHAT_ID` | — | ID чата (если пусто — работает во всех) |
| `LLM_API_KEY` | — | Ключ API (OpenAI-совместимый) |
| `LLM_BASE_URL` | https://api.deepseek.com | Адрес API |
| `LLM_MODEL` | deepseek-chat | Модель |
| `JOKES_FILE` | ./data/jokes.json | Путь к файлу с анекдотами |
| `JOKE_INTERVAL` | 100 | Интервал шуток в сообщениях |
| `PROBE_WINDOW` | 10 | Окно попыток |
| `QUALITY_THRESHOLD` | 7 | Порог качества |
| `DEDUP_WINDOW` | 50 | Окно дедупликации |
